# 구독 누수 방지 매니저

## 한줄 요약
> 한국 카드/결제 시스템에 맞춤화된 구독서비스 통합 관리 앱으로, 정기결제 자동 추적·만료일 알림·월별 구독비 분석·미사용 구독 경고를 통해 구독 누수를 방지한다.

## 해결하려는 문제 (Pain Point)
- **구독 서비스 과다 가입**: MZ세대 1인당 평균 월 구독료 5~10만원 수준, OTT·음악·AI 도구 등 중복 가입 빈번 (네이버 블로그 "생생 MZ 톡" 2025.05 - "한 달 구독료 10만원?" MZ세대 구독 실태 보도)
- **무료 체험 후 자동 결제 함정**: "밀리의 서재 1달 무료체험 후 자동 결제가 되어서" (네이버 지식iN), "무료 체험 후 유료로 전환되는 앱은 미리 관리하지 않으면 낭비" (네이버 블로그 "제대로하는테크")
- **해지 방법 복잡**: "정기결제 취소가 안 돼요" (네이버 지식iN), 앱마다 해지 경로가 다르고, 해외 서비스는 아이디/비밀번호 분실 시 해지 불가
- **지출 파악 어려움**: "이게 왜 결제됐지?" (네이버 블로그 "마커키퍼"), 여러 카드에 분산된 정기결제를 한눈에 파악하기 어려움
- **구독 피로감**: "부담스러운 구독시대" (네이버 카페 "스타우브 홀릭"), "안쓰는 워드랑 엑셀도 구독, 포토샵도 구독, OTT만 4~5개" 등 현실적 불만

## 타겟 사용자
- **주요 타겟**: 다수의 구독 서비스를 이용하는 20~30대 직장인, 자취 1인 가구
- **연령대**: 20~40대
- **규모 추정**: 한국 OTT 가입자 약 1,700만명(중복 포함), 디지털 구독 서비스 이용자 추정 2,000만명+

## 핵심 기능 (MVP 범위, 최대 3개)
1. **구독 서비스 수동 등록 및 관리**: 사용 중인 구독 서비스를 수동으로 등록하고, 결제일·금액·결제수단을 기록. 카테고리별(OTT, 음악, 생산성, AI 등) 분류 및 월별/연별 총 구독비 대시보드 제공
2. **결제일 알림 및 만료 예고**: 결제일 D-3, D-1 푸시 알림. 무료 체험 종료일 알림으로 자동 결제 전환 방지. 연간 구독의 갱신일 사전 알림
3. **미사용 구독 자가 진단**: 일정 기간(30일 등) 직접 "사용함" 체크를 하지 않은 구독에 대해 "이 구독 아직 쓰고 계신가요?" 리마인드 알림. 월말 구독 리포트로 총비용 및 절약 가능 금액 시각화

## 수익 모델
- **프리미엄 구독 (월 1,900~3,900원)**: 무제한 구독 등록, 상세 분석 리포트, 가족/커플 공유 기능
- **무료 버전**: 구독 5개까지 등록, 기본 알림 기능
- **제휴 수익**: 구독 서비스별 할인 정보·대안 서비스 추천 시 제휴 수수료 (장기)

## 경쟁 현황
| 서비스명 | 차이점 |
|---------|--------|
| **왓섭 (Whatssub)** (한국) | 구독 관리 전용 앱. 마이데이터 본허가 보유하여 카드·계좌·네이버페이 자동 연동. 앱 내 구독 해지 대행 기능 제공. 사용자 무료(B2B 수익). VC 투자 유치(끌림벤처스, 소프트뱅크벤처스, KB인베스트먼트) |
| **뱅크샐러드** (한국) | 마이데이터 기반 카드 내역에서 정기결제 자동 추출. 구독 전용 기능은 부가 기능 수준이며, 해지 가이드나 미사용 경고는 미제공 |
| **토스** (한국) | '정기결제 조회' 기능으로 카드/계좌 기반 구독 자동 감지. 그러나 구독 관리 전문 기능(카테고리 분류, 미사용 추적, 절약 리포트)은 없음 |
| **Rocket Money** (미국) | 은행 연동 기반 구독 자동 감지 + 대행 해지 서비스. 500만+ 회원. 한국 미진출, 한국 결제 시스템 비호환 |
| **Bobby/Subby** (글로벌) | 수동 등록 방식 구독 트래커. 한국 로컬라이징 없음 (원화, 한국 서비스 DB 미지원) |

### 경쟁 리스크 (핵심)
1. **왓섭이 이미 한국형 Rocket Money 포지션을 점유**: 마이데이터 인허가 기반으로 자동 구독 탐지 + 해지 대행을 무료로 제공 중. 수동 등록 앱이 이를 대체할 이유가 없음.
2. **뱅크샐러드·토스가 마이데이터 기반으로 정기결제 조회를 무료 제공** 중이며, 이들이 구독 관리 기능을 강화할 경우 독립 앱의 존립 근거가 더 약해짐.
3. 한국에서 별도 '구독 관리 전용 앱'이 대규모 성공한 사례가 아직 없는 점도 리스크.

## 기술 구현 검토
- **React Native + TypeScript**: 수동 등록 + 알림 + 대시보드 앱이므로 기술적으로 충분히 구현 가능
- **Supabase**: 사용자별 구독 데이터 저장, 알림 스케줄링(Edge Functions + cron), RLS 정책 적용 가능
- **외부 서비스 의존도**:
  - 푸시 알림: Firebase Cloud Messaging (무료)
  - **마이데이터/오픈뱅킹 연동 불가**: 마이데이터 사업자 등록이 필요하며(금융위원회 인허가), 1인 개발자가 취득 불가능. 따라서 **카드 내역 자동 추적은 구현 불가**하며, 수동 등록 방식으로 제한됨
  - 한국 구독 서비스 DB: 직접 구축 필요 (네이버, 카카오, OTT 등 주요 서비스 정보)

### 기술적 한계 (핵심)
마이데이터 연동이 불가능하므로 **모든 구독을 사용자가 수동으로 입력**해야 한다. 이는 토스/뱅크샐러드의 자동 감지 대비 UX가 크게 열위이며, 사용자 이탈의 주요 원인이 될 수 있다.

## 판정
- **결과**: 탈락
- **탈락 단계**: 2단계 (사업성/기술/경쟁력 평가)
- **미달 항목**: 경쟁 강도 (1/5), 수익 모델 명확성 (2/5)
- **사유**: 왓섭(Whatssub)이 마이데이터 본허가 기반으로 자동 구독 탐지+해지 대행을 무료 제공 중이며, 토스(2,000만 MAU)·뱅크샐러드도 정기결제 조회를 무료 제공. 마이데이터 인허가 없이 수동 입력 방식으로는 기존 앱 대비 UX가 근본적으로 열위이고, 경쟁자가 무료인 시장에서 유료 수익화도 비현실적. Pain Point와 시장 자체는 명확하나, 1인 개발 독립 앱이 진입할 수 있는 경쟁 구조가 아님.
