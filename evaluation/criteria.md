# 평가 기준

## 점수 체계 (각 항목 1-5점, 총 30점 만점)

### 1. MVP 출시 가능성 (1-3개월)
| 점수 | 기준 |
|------|------|
| 5 | 1개월 내 핵심 기능 구현 가능 |
| 4 | 2개월 내 구현 가능 |
| 3 | 3개월 내 구현 가능 |
| 2 | 3개월 초과 예상 |
| 1 | 1인 개발로 MVP 자체가 어려움 |

### 2. Pain Point 명확성
| 점수 | 기준 |
|------|------|
| 5 | 커뮤니티에서 반복적으로 언급되는 불만 |
| 4 | 특정 그룹에서 명확한 수요 확인 |
| 3 | 논리적으로 수요가 추론됨 |
| 2 | 있으면 좋지만 급하지 않은 문제 |
| 1 | 만든 사람만 쓸 것 같음 |

### 3. 기술 스택 적합성
| 점수 | 기준 |
|------|------|
| 5 | Expo + RN + Supabase만으로 완전 구현 |
| 4 | 소수 외부 API 연동 필요 (지도 등) |
| 3 | 결제/인증 등 외부 서비스 필요 |
| 2 | 네이티브 모듈 또는 복잡한 외부 연동 필요, 또는 Windows 개발 환경에서 테스트·디버깅이 제한되는 플랫폼 전용 API 의존 |
| 1 | RN으로 구현 부적합 (AR, 고성능 그래픽 등) 또는 macOS 빌드 환경 필수 |

### 4. 시장 규모
| 점수 | 기준 |
|------|------|
| 5 | 한국 전체 성인 대상 (수천만) |
| 4 | 대형 세그먼트 (직장인, 대학생 등) |
| 3 | 중형 세그먼트 (특정 직군, 취미 등) |
| 2 | 소형 니치 (수만 명 수준) |
| 1 | 극소수 대상 |

### 5. 경쟁 강도
| 점수 | 기준 |
|------|------|
| 5 | 직접 경쟁자 없음 |
| 4 | 경쟁자 있으나 명확한 차별점 가능 |
| 3 | 경쟁자 있고 차별점이 애매함 |
| 2 | 대기업(네이버/카카오) 유사 서비스 존재 |
| 1 | 네이버/카카오가 핵심 기능으로 제공 중 |

### 6. 수익 모델 명확성
| 점수 | 기준 |
|------|------|
| 5 | 사용자가 자연스럽게 결제할 구조 |
| 4 | 프리미엄/구독 모델 적용 가능 |
| 3 | 광고 기반 수익 가능 |
| 2 | 수익화 경로가 불분명 |
| 1 | 수익화 방법을 모르겠음 |

## 판정 기준
| 총점 | 판정 |
|------|------|
| 24-30 | **채택** - 즉시 기획 착수 |
| 18-23 | **보류** - 추가 조사 후 재평가 |
| 12-17 | **탈락** - 사유 기록 후 아카이브 |
| ~11 | **탈락** - 근본적 제약 있음 |
