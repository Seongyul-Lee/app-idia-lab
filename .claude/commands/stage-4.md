CLAUDE.md의 규칙을 준수하며 4단계 PRD 정합성 판별을 수행한다.

대상 아이디어: $ARGUMENTS

## 목적
작성된 PRD를 CLAUDE.md의 "PRD 정합성 판별 기준"에 따라 사업성·리스크·기술·문서 품질 관점에서 다각도 검증한다.

## 입력
- `ideas/NNN-아이디어명-prd.md` — PRD 문서
- `ideas/NNN-아이디어명.md` — 평가 완료 아이디어 문서
- `research/competitors/NNN-경쟁분석.md` — 2단계 경쟁사 분석 (사업성 참조용)
- `research/market-data/NNN-시장분석.md` — 2단계 시장 분석 (사업성 참조용)

## 사용 도구
| 도구 | 역할 | 검증 영역 |
|------|------|----------|
| `/sc-spec-panel --mode critique` | 10인 명세 전문가 체계적 리뷰 | 문서 품질 (내적 일관성, 완결성, 측정 가능성) |
| `architect-review` (에이전트) | 클린 아키텍처/DDD 관점 설계 리뷰 | 기술 스택 정합성, 아키텍처 정합성, MVP 범위 적정성 |

> **사업성·리스크 검증**은 2단계 산출물(경쟁분석, 시장분석)을 참조하여 PRD와의 정합성을 확인한다. `/sc-business-panel`을 재실행하지 않는다.

## 절차
1. PRD 및 2단계 산출물 읽기
2. `/sc-spec-panel --mode critique`로 PRD 명세 품질 리뷰
3. `architect-review` 에이전트로 기술 설계 리뷰
4. 2단계 산출물 기반으로 PRD의 사업성·리스크 정합성 확인
5. 세 영역의 결과를 종합하여 판별 보고서 작성
6. 판별 결과를 사용자에게 보고

## 검증 체크리스트
- [ ] **사업성**: PRD 내용이 2단계 시장분석·경쟁분석과 일치하는지, 차별점·수익화·TAM/SAM/SOM 정합성
- [ ] **리스크**: 핵심 리스크 식별·대응, 외부 의존성 리스크
- [ ] **기술**: RN+TS+Supabase 구현 가능, 1인 개발 규모 적합, 3개월 MVP 범위
- [ ] **문서 품질**: 목표·기능·기술 명세 간 모순 없음, 누락 섹션 없음, KPI 측정 가능

## 전환
- 모든 검증 영역 통과 → "5단계: `/stage-5 NNN-아이디어명 채택`" 안내
- 수정 가능한 미달 사항 → "3단계 회귀: `/stage-3 NNN-아이디어명`" 안내
- 근본적 사업성/기술 결함 → "2단계 회귀: `/stage-2 NNN-아이디어명`" 안내
- 수정 불가한 근본적 결함 → "5단계: `/stage-5 NNN-아이디어명 탈락`" 안내 (사용자 동의 필수)
