# 복약케어

## 한줄 요약
> 부모님/어르신의 약 복용을 자녀가 원격으로 챙길 수 있는 가족 연동 복약관리 앱

## 해결하려는 문제 (Pain Point)

### 1. 어르신 복약 미순응 문제
- 65세 이상 만성질환자의 약 50%가 처방대로 약을 복용하지 않음 (글로벌 통계, DailyOK 인용)
- 네이버 지식iN에서 "부모님 약 관리" 관련 질문 다수: "60년생 아빠 고혈압약, 62년생 엄마 골다공증약 복용 관리" 등 자녀들의 구체적 고충 확인
- 복약 미순응으로 인한 재입원, 건강 악화가 사회적 비용으로 이어짐 (미국 기준 연간 1,000~3,000억 달러)

### 2. 기존 앱의 한국 어르신 UX 부재
- 파프리카케어: 처방전 촬영으로 약 등록 가능하나, **가족 공유/원격 모니터링 기능이 핵심이 아님**. 2024년 라이프시맨틱스에 인수됨 (누적 23만 가입자)
- Medisafe: 글로벌 1위 복약 앱. Medfriend 기능으로 가족 알림 지원하나 **한국어 미지원**, 한국 의약품 DB 미연동
- 와플랫 체크: 복약알림 + 스트레스 자가검사. 심플하지만 **가족 연동 기능 없음**
- MyTherapy: 독일 기반, 한국어 일부 지원. 가족 알림은 있으나 **한국 처방전/약봉투 인식 불가**
- 투데이이코노믹 기자 리뷰: "파프리카케어 알림 메시지 소리가 강하지 않고, 다른 앱에 묻혀 시간이 지나 남아있는 메시지로 확인하기 일쑤" — 알림만으로는 부족하다는 pain point 확인

### 3. 자녀의 원격 돌봄 니즈
- 네이버 블로그: "혼자 계신 80대 부모님, 가장 중요한 건 '일상관리'" — 약 복용, 식사, 수면 등 기본 관리가 핵심
- 카페 글: "갤럭시 헬스 앱에서 약 복용 알림 설정 가능하지만, 부모님 약을 자녀가 원격으로 확인하기 어렵다"
- 지식iN: "뇌졸중 약 복용 누락이 자주 일어나면 복용 시간 알람, 약 복용 기록 앱을 활용하라" — 전문가도 앱 사용 권고하나 적합한 한국 앱 부재

### 검색 트렌드 근거
- 네이버 DataLab: "약 복용 알림 / 복약 관리 앱 / 부모님 약 관리" 키워드 — 2025년 8월 2.2 → 9월 47.6 → 11월 84.9 → 2026년 1월 32.9. **급등 후 안정적 유지 단계**

## 타겟 사용자
- **주요 타겟**: 부모님이 만성질환 약을 복용 중인 30~50대 자녀 (설정·관리 주체)
- **보조 타겟**: 직접 약을 복용하는 60~80대 어르신 (알림 수신·복용 확인 주체)
- **연령대**: 30~50대 (앱 설치/설정), 60~80대 (일상 사용)
- **규모 추정**: 65세 이상 인구 약 950만 명 (2025년 기준), 만성질환 유병률 80% 이상 → 잠재 대상 약 760만 명. 이를 관리하는 자녀 세대까지 포함하면 1,500만+ 잠재 사용자

## 핵심 기능 (MVP 범위, 최대 3개)

### 1. 간편 약 등록 + 복약 알림
- 약 이름 검색(식약처 의약품 DB 활용) 또는 직접 입력으로 복용 약 등록
- 복용 시간·횟수·기간 설정 → 푸시 알림 + 큰 글씨/큰 버튼 UI로 "복용 완료" 탭
- 어르신 전용 대형 UI 모드 (큰 글씨, 높은 대비, 최소 탭 수)

### 2. 가족 연동 + 미복용 알림
- 초대 코드/QR로 가족 그룹 생성 (최대 5명)
- 어르신이 알림 후 일정 시간 내 "복용 완료"를 누르지 않으면 → 자녀에게 미복용 푸시 알림 전송
- 자녀 앱에서 부모님의 복약 현황 대시보드 (오늘/주간 복약률, 미복용 이력)

### 3. 복약 기록 + 리포트
- 일별/주별/월별 복약 이력 캘린더 뷰
- 복약 순응도(%) 자동 계산 및 트렌드 그래프
- 병원 방문 시 의사에게 보여줄 수 있는 복약 리포트 PDF 내보내기

## 수익 모델
- **프리미엄 구독 (월 3,900원~)**: 가족 연동 2명 초과 시, 상세 리포트, PDF 내보내기, 다중 프로필 관리
- **기본 무료**: 1인 복약 알림 + 가족 연동 1명까지 무료 (충분한 사용자 확보 후 전환)
- **(향후)** 약국/건강기능식품 파트너 광고, 복약 데이터 기반 헬스케어 서비스 연계

**핵심 차별점**: "한국 의약품 DB + 어르신 전용 대형 UI + 가족 연동 미복용 알림"의 조합은 국내에 부재

## 기술 구현 검토
- **Expo Managed Workflow + React Native + TypeScript**: 구현 가능. 푸시 알림(Expo Push Notifications → FCM/APNs), 로컬 알림 스케줄링(expo-notifications), 대형 UI 컴포넌트 모두 Expo로 구현 가능
- **Supabase**: 사용자/가족 그룹/복약 기록 CRUD, RLS로 가족 그룹 내 데이터 공유, Edge Functions로 미복용 체크 + 가족 알림 트리거 가능. Realtime으로 복약 상태 즉시 반영
- **외부 서비스 의존도**:
  - 식약처 의약품 공공 API (무료, 의약품 정보 조회)
  - Expo Push Notifications (FCM/APNs 자동 연동, 무료, 푸시 알림)
  - 결제: 초기 MVP에서는 불필요, 프리미엄 전환 시 인앱 결제 연동
  - 본인인증/지도: 불필요
- **1인 개발 적합성**: 높음. 핵심은 CRUD + 알림 스케줄링 + 가족 그룹 로직으로, 복잡한 외부 연동 최소화
