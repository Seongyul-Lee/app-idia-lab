# 데일리셀프 (DailySelf)

## 한줄 요약
> 매일 아침 컨디션(에너지, 수면, 기분)을 입력하면, 오늘에 맞춤화된 루틴+셀프케어를 제안하는 적응형 일일 플래너

## 해결하려는 문제 (Pain Point)
- 기존 루틴/습관 앱은 "매일 같은 계획"을 강요하지만, 실제 컨디션은 날마다 다름
- 한국 직장인은 야근, 회식, 수면 부족 등으로 에너지가 유동적 → 고정 루틴 실패 → 자책 반복
- 루틴 앱(루티너리, 챌린저스)은 "실행"만, 감정 앱(하루콩)은 "기록"만 → 컨디션과 행동을 연결하는 앱 부재
- 작심삼일의 핵심 원인: 컨디션이 안 좋은 날에도 동일한 목표를 강요받는 구조
- 2026 레디코어 트렌드: 체계적 자기관리 시스템에 대한 수요 급증, 하지만 "유연한 체계"를 제공하는 서비스 없음

## 타겟 사용자
- **주요 타겟**: 자기관리를 하고 싶지만 작심삼일로 실패하는 한국 직장인/대학생
- **연령대**: 20대 중반 ~ 30대 후반 (MZ세대)
- **규모 추정**: 한국 성인 루틴 보유율 80% (10명 중 8명), 6개월 내 이탈율 48% → 루틴 실패 경험자 약 1,500만 명 이상

## 핵심 기능 (MVP 범위, 최대 3개)
1. **아침 체크인 (30초)**: 에너지 레벨(1-5), 수면 시간, 오늘 기분(좋음/보통/나쁨) 입력. 간단한 슬라이더+탭 UI
2. **오늘의 적응형 플랜**: 체크인 결과 기반으로 루틴 난이도 자동 조절. 에너지 낮으면 "5분 스트레칭+차 한잔" 제안, 높으면 "30분 운동+독서 30분" 제안. 사용자가 등록한 루틴 풀에서 컨디션에 맞게 선별·재배치
3. **저녁 회고 + 주간 리포트**: 실제 달성 기록 + 한줄 회고. 주간 컨디션-달성률 상관관계 통계. "에너지가 3 이상인 날 달성률이 85%입니다" 같은 인사이트

## 수익 모델
- **프리미엄 구독** (월 3,900원 / 연 29,900원):
  - 무료: 기본 루틴 5개, 주간 리포트
  - 유료: 무제한 루틴, AI 개인화 인사이트, 월간 트렌드 분석, 커스텀 루틴 카테고리, 셀프케어 콘텐츠 라이브러리

## 경쟁 현황
| 서비스명 | 차이점 |
|---------|--------|
| 루티너리 | 고정 시간+타이머 기반. 컨디션 적응 없음. 해빗 스태킹에 특화 |
| 챌린저스 | 금전적 동기부여만 제공. 컨디션 고려 없음. 85% 미달 시 금전 손실 |
| 투두메이트 | 소셜 할 일 관리. 루틴 시간/에너지 적응 없음 |
| 오늘의루틴 | 심플 체크리스트. 적응형 기능 없음 |
| 루빗 (Roubit) | 습관+감정 통합, ChatGPT 멘탈케어 도입, 100만 DL. 그러나 컨디션 기반 루틴 적응/재배치 기능 없음. 게이미피케이션 중심 |
| 하루콩 | 감정 기록만. 루틴/행동 연결 없음 |
| 마이루틴 (MyRoutine) | 루틴+투두 통합 관리, 150만 사용자, 23억 투자 유치. 신호등 시각화. 컨디션 적응 기능 없음 |
| 앳플리 (Atflee) | 감정+식단+운동+명상 통합 시도. 가장 유사한 포지션이나 시장 인지도 낮음, 감정-루틴 인과 분석 부재 |
| 데일리셀프 (본 앱) | **컨디션 입력 → 적응형 루틴 제안**. 습관+감정 교차. 실패하지 않는 유연한 구조. 루빗/앳플리 대비 "컨디션 인식 → 난이도 자동 조절"이 핵심 차별점 |

## 기술 구현 검토
- **React Native + TypeScript**: 완전 구현 가능. 슬라이더/탭 UI, 푸시 알림, 캘린더 뷰 등 표준 컴포넌트로 충분
- **Supabase**: 사용자 루틴 DB, 체크인 기록, 통계 데이터 저장. PostgreSQL 함수로 통계 계산 가능. Row Level Security로 사용자별 데이터 격리
- **외부 서비스 의존도**: 낮음. 푸시 알림(Expo Push), 인앱결제(RevenueCat) 정도. 핵심 로직은 규칙 기반으로 외부 AI API 없이 구현 가능. 향후 LLM API로 개인화 인사이트 보강 가능

## 판정
- **결과**: `채택` (24점 이상)
- **사유**: 총점 25점으로 채택 기준(24점) 충족. Pain Point가 데이터로 강력히 검증됨. "컨디션 적응형 루틴"은 경쟁사 6개 모두에 없는 명확한 차별점. 기술 실현성 최상(RN+Supabase만으로 구현, 2개월 MVP). 수익 모델과 경쟁 강도에서 리스크 존재하나, 차별점과 Pain Point의 강도가 이를 상쇄.
