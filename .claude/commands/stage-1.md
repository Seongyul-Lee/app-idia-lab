CLAUDE.md의 규칙을 준수하며 1단계 아이디어 발굴을 수행한다.

$ARGUMENTS

## 목적
한국 시장의 실제 pain point를 기반으로 앱 아이디어를 도출한다.

## 사용 도구
| 도구 | 역할 |
|------|------|
| `/sc-brainstorm` | 소크라테스식 대화로 아이디어 체계적 발굴. `--depth deep`으로 심층 탐색 가능 |
| `/sc-research --depth standard` | 한국 시장 트렌드, pain point 웹 리서치 |
| MCP: `naver-search` | 네이버 블로그/카페/뉴스에서 한국 사용자 불만·수요 수집 |
| MCP: `brave-search` | 글로벌 트렌드 및 해외 유사 앱 탐색 |

## 절차
1. 탐색 주제 설정 (사용자 지정 또는 트렌드 기반)
2. `/sc-brainstorm`으로 아이디어 발굴 대화 진행
3. MCP 검색으로 pain point 실재 여부 확인
4. 제외 기준(법적 인허가, 기업 협약 필수, 1인 개발 불가) 사전 필터링
5. `ideas/_template.md` 기반으로 아이디어 초안 작성 (평가 점수 섹션 제외)

## 산출물
| 파일 | 위치 |
|------|------|
| 아이디어 초안 | `ideas/NNN-아이디어명.md` (평가 점수 미기입) |
| 트렌드 메모 (선택) | `research/market-data/` |

## 전환
- 초안 완료, 제외 기준 미해당 → 사용자에게 "2단계: `/stage-2 NNN-아이디어명`" 안내
- 제외 기준 해당 → 폐기 후 새 아이디어 발굴
- 사용자가 종료 요청 → 중단
